# How to Present This Project in Interview

## 30-second version
I built a self-initiated data quality project to simulate ETL testing in a healthcare reporting scenario. I created source and target datasets, wrote SQL test cases for count checks, null checks, duplicates, business rules, and reconciliation, and documented the defects in a structured log.

## 60-second version
I wanted to demonstrate practical Data QA capability, so I created a healthcare-style project with source and transformed target data. I intentionally introduced data issues such as duplicates, nulls, negative cost values, and mismatches between source and target. Then I wrote SQL validation queries to detect those issues, created test cases, produced a defect log, and summarized the outcomes in a small dashboard. This helped me show how I would validate data before it is used in reporting or BI.

## What this proves
- Understanding of data quality dimensions
- ETL testing mindset
- SQL testing capability
- Ability to document and communicate defects
