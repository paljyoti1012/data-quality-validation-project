# Project Summary

## Objective
Build a self-initiated data quality and ETL testing project to validate source-to-target data movement for a healthcare reporting dataset.

## Scope
This project demonstrates:
- SQL-based validation
- source-to-target reconciliation
- duplicate checks
- null checks
- business rule validation
- structured defect logging
- test case documentation
- simple evidence reporting

## Dataset
A healthcare-style patient treatment dataset was used with source and target layers.

## Validation Results
- Source record count: 8
- Target record count: 8
- Duplicate issues: 1
- Null mandatory field issues: 2
- Invalid cost issues: 2
- Reconciliation failures: 3
- Missing records in target: 1
- Field-level mismatches: 1

## Outcome
The project simulates real Data QA activity and can be used in a CV, GitHub portfolio, and interviews to demonstrate practical testing capability.
